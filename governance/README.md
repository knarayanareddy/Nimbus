# ClimaFi Governance Framework (Future)

## Overview

ClimaFi will transition from centralized admin control to a **decentralized governance model** after mainnet launch.

## Proposed Governance Structure

### Phase 1 (Current)
- Admin-controlled (single multisig)
- Emergency pause authority
- Parameter changes require off-chain coordination

### Phase 2 (Post-Mainnet)
- **ClimaFi DAO** with governance token
- On-chain proposals for:
  - Protocol fee changes
  - New region approvals
  - LTV limit adjustments
  - Oracle provider whitelisting

### Phase 3 (Advanced)
- Risk parameter automation via on-chain oracles
- Reinsurance pool governance
- Cross-chain expansion voting

## Security Considerations

- All governance actions will require timelock (minimum 48 hours)
- Critical actions (pause, fee changes) will require higher quorum
- Emergency multisig retained for 6–12 months post-launch

## Implementation Roadmap

- Q3 2026: Governance token design
- Q4 2026: On-chain proposal system
- 2027: Full DAO transition

*This is a high-level plan and subject to change based on community input and regulatory developments.*
